## Prerequisites

Before setting up BuddyPie, ensure you have the following:

| Requirement | Version | Notes |
|-------------|---------|-------|
| Node.js | 22+ | LTS recommended |
| Bun | Latest | Package manager and runtime |
| Git | Any | For repository operations |

You'll also need accounts with:

- **Convex** - Backend database and functions
- **Clerk** - Authentication provider
- **Daytona** - Sandbox infrastructure
- **OpenRouter or Venice** - AI model providers (at least one)

## Installation

### 1. Clone and Install

```bash
git clone https://github.com/your-org/buddypie2.git
cd buddypie2
bun install
```

### 2. Set Up Convex

Create a Convex deployment at [convex.dev](https://convex.dev):

```bash
# Log in to Convex
bunx convex login

# Create a new deployment
bunx convex deploy --preview
```

Note your deployment URL for the environment configuration.

### 3. Configure Clerk

Create a Clerk application at [clerk.com](https://clerk.com):

1. Create a new application
2. Enable the authentication methods you need
3. Configure the JWT template for Convex

The JWT issuer domain format is: `https://<your-app>.clerk.accounts.dev`

### 4. Get Daytona API Key

Sign up at [daytona.io](https://daytona.io) and generate an API key from your dashboard.

## Environment Variables

Create `.env.local` in the project root:

```bash
# Clerk Authentication
CLERK_PUBLISHABLE_KEY=pk_test_xxxxx
CLERK_SECRET_KEY=sk_test_xxxxx
CLERK_JWT_ISSUER_DOMAIN=https://your-app.clerk.accounts.dev

# Convex
VITE_CONVEX_URL=https://your-deployment.convex.cloud
CONVEX_SITE_URL=https://your-deployment.convex.site

# Daytona
DAYTONA_API_KEY=dtn_xxxxx
DAYTONA_API_URL=https://app.daytona.io/api

# AI Model Providers (at least one required)
OPENROUTER_API_KEY=sk-or-v1-xxxxx
VENICE_API_KEY=venice_xxxxx
```

### Required Variables

| Variable | Description |
|----------|-------------|
| `CLERK_PUBLISHABLE_KEY` | Frontend Clerk key |
| `CLERK_SECRET_KEY` | Backend Clerk secret |
| `CLERK_JWT_ISSUER_DOMAIN` | Clerk JWT issuer URL |
| `VITE_CONVEX_URL` | Convex deployment URL |
| `DAYTONA_API_KEY` | Daytona API authentication |

### Optional Variables

| Variable | Description |
|----------|-------------|
| `DAYTONA_API_URL` | Override Daytona API endpoint |
| `CONVEX_SITE_URL` | Explicit Convex HTTP actions URL |
| `OPENROUTER_API_KEY` | Enable OpenRouter models |
| `VENICE_API_KEY` | Enable Venice AI models |

## Running the Application

### Development Mode

```bash
# First time: push Convex schema
bunx convex dev --once

# Start development server
bun run dev
```

The `dev` script runs:
1. Convex schema sync once
2. Vite dev server (frontend)
3. Convex dev sync (hot reload)

### Production Build

```bash
bun run build
bun run start
```

## Convex Configuration

### Auth Configuration

The Convex auth config (`convex/auth.config.ts`) must match your Clerk issuer:

```ts
import { defineAuthConfig } from 'convex/server'

export default defineAuthConfig({
  providers: [
    {
      domain: process.env.CLERK_JWT_ISSUER_DOMAIN!,
      applicationID: 'convex',
    },
  ],
})
```

### Environment Variables in Convex

Set these in the Convex dashboard under Settings > Environment Variables:

- `X402_PAY_TO_ADDRESS` - Wallet address for x402 settlement
- `X402_EIP712_TOKEN_NAME` - USDC token name (if overriding)
- `X402_EIP712_TOKEN_VERSION` - USDC token version

## Verifying Setup

### Check Authentication

1. Open the application in your browser
2. Sign in with Clerk
3. Verify you see the dashboard

### Test Sandbox Launch

1. Enter a public GitHub repository URL
2. Select a preset (e.g., "General Engineer")
3. Leave the kickoff prompt empty to use the default
4. Click Launch

The sandbox creation takes 1-3 minutes. You'll see status updates as:
1. Creating the Daytona sandbox
2. Cloning the repository
3. Installing OpenCode
4. Seeding the initial session

## Troubleshooting

<Accordions>
  <Accordion id="convex-auth-error" title="Convex authentication errors">
    
    Ensure your Clerk JWT template includes `convex` as the audience and the issuer domain matches `CLERK_JWT_ISSUER_DOMAIN`.
    
    Check the Convex auth logs in the dashboard for specific error messages.
    
  </Accordion>
  
  <Accordion id="daytona-api-error" title="Daytona API connection issues">
    
    Verify your `DAYTONA_API_KEY` is valid and not expired. Check that `DAYTONA_API_URL` is correct if you're using a custom endpoint.
    
    The default URL is `https://app.daytona.io/api`.
    
  </Accordion>
  
  <Accordion id="model-provider-error" title="AI model provider errors">
    
    At least one of `OPENROUTER_API_KEY` or `VENICE_API_KEY` must be set.
    
    The `general-engineer` and `frontend-builder` presets require OpenRouter. The `docs-writer` preset requires Venice.
    
  </Accordion>
  
  <Accordion id="sandbox-timeout" title="Sandbox creation timeout">
    
    Sandbox creation has a 5-minute timeout. Large repositories or slow networks may exceed this.
    
    Check the sandbox status in the Convex dashboard. Failed sandboxes are automatically cleaned up.
    
  </Accordion>
</Accordions>
